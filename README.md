# Estimate Engineering Team

Команда агентов для подготовки строительных смет ремонта квартир по проектным PDF.

Проект заточен под один сценарий: на вход подаются PDF-файлы дизайн-проекта или рабочей документации квартиры, на выходе формируется Excel-смета с вкладками работ, материалов, исходных объемов, допущений и проверки. Цены агенты не заполняют: колонки для цены и суммы создаются, но цена за единицу остается для ручного ввода после согласования с рабочими и поставщиками.

## Для каких задач

Проект нужен, когда есть проект квартиры с листами вроде:

- визуализации;
- план монтажа конструкций и перегородок;
- план расстановки мебели;
- план расстановки сантехнического оборудования;
- план расстановки светильников;
- план расстановки электротехнического оборудования;
- схема монтажа потолков;
- экспликации, ведомости дверей, помещений, отделки, если они есть.

Основной диапазон объектов: квартиры примерно от 30 до 200 м2.

Лучший входной формат: векторные PDF, где можно выделять текст, размеры, подписи и таблицы. Со сканами проект может работать хуже и потребует OCR/ручной проверки.

## Что получается на выходе

Основной результат:

```text
agent-runtime/outputs/final-estimate.xlsx
```

Ожидаемые вкладки Excel:

- `Работы` — перечень работ, единицы измерения, объемы, формулы расчета, цена за единицу, сумма, источник объема.
- `Материалы` — черновые материалы, нормы расхода, базовый объем, расчетное количество, запас/округление, цена за единицу, сумма.
- `Исходные объемы` — все извлеченные из PDF объемы с указанием файла, страницы, листа проекта и уверенности.
- `Допущения` — явные предположения, принятые из-за неполноты проекта.
- `Проверка` — замечания аудитора: критичные ошибки, предупреждения и вопросы для ручного контроля.

Цены не заполняются автоматически. В итоговой таблице должны быть:

- колонка `Цена за ед., руб.`;
- колонка `Сумма, руб.`;
- формулы суммы строк;
- формулы итогов разделов;
- общий итог.

## Главный агент

Точка входа в проект:

```text
router-estimator
```

Он управляет всей цепочкой и отвечает за то, чтобы каждый subagent создал нужный промежуточный файл. Отдельный командный режим с несколькими равноправными участниками в этом проекте не используем: пользователь обращается к главному агенту `router-estimator`, а он сам делегирует этапы специализированным subagents Claude Code.

Пример запроса:

```text
Используй router-estimator.
По PDF проекта квартиры составь Excel-смету работ и черновых материалов без цен.
PDF лежат здесь: /path/to/project-pdfs"
```

## Цепочка агентов

```text
router-estimator
  -> pdf-takeoff-agent
  -> quantity-surveyor-agent
  -> works-estimator-agent
  -> materials-estimator-agent
  -> norms-research-agent
  -> xlsx-builder-agent
  -> estimate-auditor-agent
```

### `router-estimator`

Координатор. Принимает задачу, делегирует этапы subagents по цепочке, проверяет наличие артефактов и возвращает финальный результат.

### `pdf-takeoff-agent`

Разбирает проектные PDF:

- определяет листы проекта;
- извлекает помещения, площади, подписи, ведомости;
- считает или фиксирует объемы перегородок, потолков, полов, стен;
- собирает количество светильников, электроточек, сантехнических приборов, дверей;
- сохраняет первичный обмер.

Основные файлы:

```text
agent-runtime/shared/project-index.json
agent-runtime/shared/extracted-quantities.json
agent-runtime/shared/takeoff-summary.md
```

### `quantity-surveyor-agent`

Проверяет и нормализует объемы:

- приводит единицы к `м2`, `м3`, `пог.м`, `шт`, `компл`;
- проверяет согласованность площадей;
- отмечает отсутствующие высоты, толщины, типы конструкций;
- выносит сомнительные места на ручную проверку.

Основные файлы:

```text
agent-runtime/shared/normalized-quantities.json
agent-runtime/shared/quantity-checks.md
```

### `works-estimator-agent`

Формирует черновик вкладки `Работы`:

- разделы работ;
- наименования;
- единицы измерения;
- объемы;
- текстовые формулы расчета;
- ссылки на источники;
- пустые цены и формулы будущих сумм.

Основной файл:

```text
agent-runtime/shared/works-draft.json
```

### `materials-estimator-agent`

Формирует черновик вкладки `Материалы`:

- черновые материалы;
- нормы расхода;
- базовые объемы;
- расчетные количества;
- запас и округление;
- связь материала с работой.

Основной файл:

```text
agent-runtime/shared/materials-draft.json
```

### `norms-research-agent`

Проверяет нормативную логику ГЭСН/ФЕР:

- единицы измерения;
- состав работ;
- типовые расходные нормы;
- вспомогательные операции, которые легко забыть;
- места, где нужен точный код или ручная проверка.

Важно: агент не должен выдумывать точные коды ГЭСН/ФЕР без проверки и не должен заполнять цены.

Основной файл:

```text
agent-runtime/shared/norms-notes.md
```

Официальная точка входа для проверки федеральных сметных нормативов:

```text
https://www.minstroyrf.gov.ru/trades/tsenoobrazovanie/federalnyy-reestr-smetnykh-normativov/
```

### `xlsx-builder-agent`

Собирает итоговый Excel:

- создает вкладки;
- переносит работы и материалы;
- добавляет формулы;
- добавляет исходные объемы, допущения и проверки;
- по возможности пересчитывает и проверяет формулы.

Основной файл:

```text
agent-runtime/outputs/final-estimate.xlsx
```

### `estimate-auditor-agent`

Проверяет результат:

- нет ли пустых формул вида `=`;
- все ли суммы считаются формулами;
- все ли объемы имеют источник;
- нет ли двойного счета;
- есть ли материалы без связанных работ;
- есть ли работы без нужных черновых материалов;
- все ли сомнительные объемы вынесены на проверку.

Основной файл:

```text
agent-runtime/shared/audit-report.md
```

- считать ли мебель/двери/стеклянные перегородки как подрядные позиции.

## Как работать с ценами

Агенты не заполняют цены.

Правильный процесс:

1. Агенты формируют объемы, работы и материалы.
2. В Excel остаются пустые или нулевые цены.
3. Пользователь согласует цены с рабочими и поставщиками.
4. Пользователь вручную заполняет `Цена за ед., руб.`.
5. `Сумма, руб.` и итоги пересчитываются формулами.

Такой подход снижает риск ложной точности: агент помогает с объемами и структурой, но не подменяет реальные договоренности по стоимости.

## Что обязательно проверять вручную

Даже при хороших векторных PDF вручную нужно проверить:

- высоты потолков;
- толщины и типы перегородок;
- количество слоев ГКЛ;
- состав мокрых зон и гидроизоляции;
- электрические трассы, если на проекте есть только точки;
- сантехнические трассы, если есть только приборы;
- дверные проемы и перемычки;
- зоны, где объем получен не из ведомости, а расчетом;
- все позиции с низкой уверенностью;
- все допущения на вкладке `Допущения`.

## Ограничения

Проект не является заменой профессиональной экспертизы проектной документации.

Агенты могут ошибаться в случаях:

- PDF сканированный или низкого качества;
- размеры отсутствуют или конфликтуют;
- в проекте нет ведомостей;
- разные листы противоречат друг другу;
- электрика и сантехника показаны без трасс;
- часть работ определяется по факту на объекте;
- выбранная строительная система не указана.

Поэтому итоговая смета должна проходить ручную проверку перед отправкой рабочим или заказчику.
